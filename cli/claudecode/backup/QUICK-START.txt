TROUBLESHOOTING
================================================================================

Q: VBS not launching on boot
A: Manually test first: & "wscript.exe" "path\to\ClawdBot_Startup.vbs"
   Check logs: Get-Content ".openclaw\startup.log"
   Verify scheduled task exists: Get-ScheduledTask -TaskName "ClawdBot_Startup_Launcher"

Q: Scheduled task not created during restore
A: This is OK — restore script tries to create it but falls back gracefully
   To create manually:
   $action = New-ScheduledTaskAction -Execute "wscript.exe" -Argument "`"$env:APPDATA\Microsoft\Windows\Start Menu\Programs\Startup\ClawdBot_Startup.vbs`""
   $trigger = New-ScheduledTaskTrigger -AtLogOn
   Register-ScheduledTask -TaskName "ClawdBot_Startup_Launcher" -Action $action -Trigger $trigger -RunLevel Highest

Q: Startup.log shows errors
A: Check that OpenClaw.exe path is correct in the VBS
   Default: C:\Users\micha\.openclaw\openclaw.exe
   Edit the VBS file if path differs on new PC
